Audit question: How is technical documentation controlled, including versioning, approvals, and access?
Chunks used: 37
================================================================================
The evidence regarding how technical documentation for the Halcyon 4.0 (HAL 4.0) radiotherapy system is controlled, including versioning, approvals, and access is found in the following document excerpts:

1. **Versioning and Configuration Management:**
   - The manufacturer documents configuration items and their versions as part of the software system configuration. Configuration management is addressed in the **DND19 Software Configuration Management Procedure Rev 02**. This procedure includes documentation of configuration items and handling changes only in response to approved change requests, as seen in the LL00000016 Change Request Rev 05 [Regulatory / Test Reports - Regulatory Compliance / 104993953MIN-001 - HAL4.0 - IEC 60601-1.pdf, Page 299].

2. **Approvals:**
   - The electronic signature records are appended to the last page of the secured document, indicating approvals. For instance, the HAL4.0 Process Control Master Plan was electronically signed by the relevant personnel, reinforcing the approval process [Design Transfer / Manufacturing Documentation / D60826-ENU-02_HAL4.0 Process Control Master Plan Rev 02.pdf, Page 9].

3. **Access and Storage:**
   - The Design History File (DHF) indicates that the technical documentation is stored in the document control system known as Nucleus (Nucleus>Business Workgroups>DHF>Delivery Systems>Halcyon>4.0) [Design Output Specification / D67731 HAL 4.0 Design History File Index Rev 05.docx.pdf, Page 14].
   - Access to hardware and software is controlled in several ways, including key to the console, user rights protection, and secure logins [Regulatory / Essential Requirements Checklist / D70060 HAL 4.0-GSPR Checklist.pdf, Page 22].
   - Setting up user access roles is documented in P1039629-001-A Halcyon and Ethos Radiotherapy System IFU-RUA 4.0, detailing the distinct roles and access rights [Regulatory / Test Reports - Regulatory Compliance / D69966 HAL 4.0 Compliance Assessment of Product Technical Requirements Rev 02.pdf, Page 26-27].

These documents collectively illustrate the control mechanisms for technical documentation pertinent to versioning, approvals, and access.
