Audit question: How are product variants and configurations managed within technical documentation?
Chunks used: 37
================================================================================
The audit question asks about the management of product variants and configurations within the technical documentation of the Halcyon 4.0 radiotherapy system. The excerpts provide several relevant insights:

1. **Configuration Management Procedures**:
   - The configuration management for Halcyon 4.0 is detailed in the Design and Development Plan (DDP), which includes the documentation of configuration items and their versions. These are managed via the DND19 Software Configuration Management Procedure as cited in [Regulatory / Test Reports - Regulatory Compliance / 104993953MIN-001 - HAL4.0 - IEC 60601-1.pdf | Page: 299].

2. **Change Control and Configuration Updates**:
   - Updates to configuration items are made only in response to approved change requests, as managed by the DND19 Software Configuration Management Procedure and documented in the Design and Development Plan (DDP), as noted in [Regulatory / Test Reports - Regulatory Compliance / D101277 106485676MIN-001 IEC60601-1 Ed.3.2 HAL4.0_30_APR_2026 Rev 1.pdf | Page 286-287].

3. **BOM and Configuration Specifics**:
   - For new configurations such as Halcyon Evolve, the BOM structure needs adjusting, and the process is documented through various product management and engineering coordination strategies, as seen in [Design Changes / Maintenance Releases / D95424 HAL 4.0 MR2 (HAL Evolve) Maintenance Release Document Rev 04.pdf | Page: 6-7 & Page: 16].

4. **Software Configuration Management**:
   - The documents outline specific responsibilities and methods to handle software as part of the configuration management scheme. This is elaborated under software configuration management strategies in excerpts like [Design Changes / Maintenance Releases / D77160 HAL 4.0 Ethos with standard kV imager MRD Rev 05.docx.pdf | Page: 17-18].

5. **Product Structuring for Variants**:
   - The product structuring, involving the handling of configurational software and hardware changes, places an emphasis on maintaining a clear and structured approach to variants through global documentation and configuration setup in systems like SAP, as found in [01. Design and Development Planning / 01. System / D30786 Halcyon 2.0 Regulatory Plan Rev 1.0.doc.pdf | Page: 51-52].

These excerpts collectively describe that Halcyon 4.0's product variants and configurations are meticulously managed through established software configuration management procedures, defined change control protocols, and comprehensive documentation practices involving multi-department coordination.
