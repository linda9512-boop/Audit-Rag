The control of technical documentation for the Halcyon 4.0 radiotherapy system includes aspects of versioning, approvals, and access as follows:

1. **Versioning and Documentation Control**: The Design History File Index for Halcyon 4.0 references all required project documentation with titles, identification numbers, and revision numbers. The documentation is stored electronically in the document control system "Nucleus," ensuring proper versioning and management [Design Output Specification / D67731 HAL 4.0 Design History File Index Rev 05.docx.pdf, Page 13-14].

2. **Approvals**: Approvals for user assistance and documentation involve electronic sign-offs, as evidenced by multiple documents that have been reviewed and electronically signed by designated approvers. For instance, document D68539, which deals with the build procedure, includes electronic signatures as a part of its approval process [Design Output Specification / Build Procedures / D68539 MPC 2.37 SBP Rev 01.pdf, Page 9]. Additionally, documents undergo a structured approval process where final distribution versions are proofread, approved, and signed off before being filed in the Design History File (DHF) [Design Output Specification / Labeling / D96388 HAL 4.0 UA Legal Information (EU REP Change) UA Checklist Rev 01.docx.pdf, Page 3-4].

3. **Access Control**: After approval, the finalized documents, such as user assistance plans, are uploaded to the GPL Customer Documentation system, where they are accessible to authorized personnel [Design Output Specification / Labeling / D96388 HAL 4.0 UA Legal Information (EU REP Change) UA Checklist Rev 01.docx.pdf, Page 4-5]. Additionally, the document control system (Nucleus) ensures that only authenticated users have access to the critical documentation, further demonstrating control over who can access the information [Design Output Specification / D67731 HAL 4.0 Design History File Index Rev 05.docx.pdf, Page 13-14].
