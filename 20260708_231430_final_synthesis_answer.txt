Audit question: How are updates and revisions to technical documentation controlled?

================================================================================
The control of updates and revisions to technical documentation for the Halcyon 4.0 radiotherapy system involves several processes and procedures as evidenced by different documents:

1. **Service Documentation Updates**: There are explicit instructions to review and update service documentation as needed. For example, the service instructions related to lubrication procedures and rail configurations are updated to ensure clarity ([Risk Management / Design FMEAs / D101531 - HAL CCH Rail Vendor Change – DFMEA Design Review.docx.pdf | Page: 3]).

2. **Software Requirements Management**: The requirements management work instruction specifies the procedures for reviewing and updating requirements ([Regulatory / Applicable Standards / D68985 HAL 4.0 IEC 62304+A1 2015 Software Lifecycle Assessment Rev 01.docx.pdf | Page: 28]).

3. **Configuration Management**: Change control after release is governed by processes in place, such as the DND01 Design and Development Procedure, which addresses the evaluation and implementation of upgrades. This control extends to third-party project components as mandated by Varian’s process for contractual change management ([Regulatory / Applicable Standards / D68985 HAL 4.0 IEC 62304+A1 2015 Software Lifecycle Assessment Rev 01.docx.pdf | Pages: 61-62]).

4. **Approval and Traceability**: Before final approval, there are detailed checks and peer reviews to ensure the completeness and correctness of the updated documents. Traceability of the document updates and their approval is maintained, with the signed approval forms and documents filed in the Design History File ([Design Output Specification / Labeling / D65651 Halcyon-Ethos 4.0 IFU (P1058124-001-A) User Assistance Checklist Rev01.docx.pdf | Pages: 4-5]).

5. **Formal Review and Validation**: There is a formal review process where the accuracy, clarity, and completeness of information are verified with reviewers, which helps in maintaining the integrity of technical documentation ([Design Output Specification / Labeling / D77711 Halcyon and Ethos Radiotherapy System 4.0 IFU (P1039628-003-C) Checklist Rev 01.docx.pdf | Pages: 4-5]).

6. **Change Request and Maintenance Release**: Change requests and maintenance releases are systematically documented and reviewed, involving activities such as impact analysis and defining approval strategies ([Design Changes / Change Requests / D72997 HAL 4.0 MR1 Project Plan Change Request 1 Rev01.docx.pdf | Pages: 12-13]).

These documents collectively show a structured approach to managing updates and revisions to the technical documentation of the Halcyon 4.0 radiotherapy system.
