Based on the excerpts provided, product variants and configurations for the HAL 4.0 device are primarily managed through software configuration management and regulatory documentation. Here's how they are managed:

1. **Configuration Management:**
   The DND19 Software Configuration Management Procedure is a critical document that outlines the management of configuration items and their versions for Halcyon 4.0. It addresses the identification, documentation, change control, and overall management of configuration items, which are essential for maintaining different product configurations and variants [Regulatory / Applicable Standards / D68985 HAL 4.0 IEC 62304+A1 2015 Software Lifecycle Assessment Rev 01.docx.pdf, page 18-19, 51; Regulatory / Test Reports - Regulatory Compliance / D101277 106485676MIN-001 IEC60601-1 Ed.3.2 HAL4.0_30_APR_2026 Rev 1.pdf, page 231-232, 286-287].

2. **Design Development Plan (DDP):**
   The DND04T01 Design and Development Plan (DDP) includes details on configuration management and references procedures for managing software configurations. This ensures that different variations or configurations of the HAL 4.0 system are controlled and documented appropriately [Regulatory / Applicable Standards / D68985 HAL 4.0 IEC 62304+A1 2015 Software Lifecycle Assessment Rev 01.docx.pdf, page 76-77].

3. **Change and Version Control:**
   Changes to configuration items are controlled through approved change requests as documented in the DND04F01 Requirements or Project Plan Change Request. This aspect helps accommodate product variants by managing changes systematically and maintaining version control [Regulatory / Test Reports - Regulatory Compliance / 104993953MIN-001 - HAL4.0 - IEC 60601-1.pdf, page 240-242].

4. **Project Documentation:**
   Regulatory documentation such as the HAL 4.0 Regulatory Plan Rev 5.0 includes ISD documentation requirements, which are essential for maintaining records of product variants and configurations for regulatory submissions across different jurisdictions. The documentation updates and manages the inclusion of different product variants and any related technical changes [Regulatory / Regulatory Plans / D54978 HAL 4.0 Regulatory Plan Rev 5.0.pdf, page 37-44].

5. **Software and Documentation Updates:**
   The software versioning and documentation (such as the Instructions for Use, Quick Reference Guides) are updated and managed to reflect different configurations and variants of the Halcyon system, ensuring that documentation is accurate and complete for the specific variant or configuration [Design Output Specification / Labeling / D63983 Halcyon 4.0 UA Checklist Rev 01.pdf, page 2-3].

These processes, embedded within the documented procedures and plans, ensure that product variants and configurations of the HAL 4.0 system are managed methodically in the technical documentation.
